Simplest Example : Wireless Medium

Backdrop

  • Cellular System : Base-stations, Mobile Devices, Wireless Medium
  • Wireless Link - Communication of information from a base-station to an UE through the wireless medium
  • About 6800 (57x20x6) links for a unit transmission frame (~1ms LTE 4G).

Simulation : Wireless Cellular System

  • Propogation Channel
Modelling the dynamics of wireless medium, user mobility, terrain, propagation models
  • Device Algorithms
Transmit waveform shaping, Receiver algorithms, Interference Management, Channel quality feedback
  • Scheduling 
Resource management, co-ordination between base-stations, Fairness to users
  • Performance Evaluation 
Performance impacts on the entire system to be evaluated

Design Challenge & Philosophy

Challenges
- Needs expertise in all modules to build the system model
- Researcher focuses on improving his algorithm than writing other modules

Philosophy
- Modularity : Standardized interfaces to functionalities ( 3rd party too), non-blocking implementation
- Easiness : Front-end to interconnect modules and generate code
- Scaling : Scale simulations , portability to cloud. (e.g. Overlay Unlicensed LTE and WiFi, Small-cell overlay )
- Interactability : Ability to respond to incoming commands, tunability of functional parameters (e.g. Control of transmit power based on upper layer module)

the Go way

  • Identify concurrent functionalities : go routines
⏩ All transmitters Synced Only at transmission level 
⏩ All Transmitters operate “concurrently"
⏩ Within each transmitter, there are modules connected in series and/or parallel like scheduler,
  FEC, RF module etc., Which do operate independently based on their inputs data
  • Communicate data between modules : go chan
  • Pass data, meta info containing commands through modules : go chan of struct

Chipset : A Go-interface

  • represents a interfaceable, interactable module and sub-modules
  • Create a new chipset by combining various chipsets (e.g. create a CDMA transmitter combining Spreading & Modem objects) 
type ChipSet interface {
    InPinCount() int
    OutPinCount() int
    PinByName(string) PinInfo   
    SetJson(json []byte) // Json formatted parameters for the Chip
    ModuleByName(string) ModuleInfo
    PinIn(indx int) PinInfo
    PinOut(indx int) PinInfo
    ModulesCount() int
    Module(int) ModuleInfo
}

Connecting Chipsets

  • Each module reads (scalar/vector) data at input pins and writes to output pins
  • Communicates using go-chan

Complete Cellular system

Scalable Framework

  • Simulating larger systems => More transmitter circuits, more interference links
  • Not all Transmitters affect all receivers
  • Control and synchronize the transmissions which affect the receivers
  • Managing transmission is done by a close-loop Tx and Rx buffer Managers
  • Enables spinning up additional VM's when more transmitters needed to simulate

Transmitter/Reciever Buffers

type TransmitterBuffer struct {
    source gocomm.Complex128AChannel
    sync.Mutex
    data    gocomm.SComplex128AObj    
    state   BufferState        // { DataReady |  DataSent | DataACKed }
}

type RecieverBuffer struct {
    source gocomm.Complex128AChannel
    sync.Mutex
    counter int
    data    gocomm.SComplex128AObj 
    TotalTx int        // #ofTransmitters that affects this receiver
}

BufferManager : Synchronize Signals and Interferences at the receiver

type TransmitterBufferManager struct {
    txInputBuffer map[int]*TransmitterBuffer
    feedbackTx2Rx,feedbackRx2Tx chan int
}

type ReceiverBufferManager struct {
    rxOutputBuffer map[int]*RecieverBuffer
    feedbackTx2Rx,feedbackRx2Tx  chan int
    sync.Mutex
    TxReadyStatus map[int]*Status
}
